# Stakeholder update workflow

Produces `_pdf-output/engagements/{active}/05-governance/stakeholder-updates/<YYYY-MM-DD>-<role-slug>.md` — one file per selected role archetype.

**Personalisation model:** by role archetype (per Helena's principle 5 — fidelity, not flattery). The user selects one or more archetypes from the audience pool. The workflow generates one file per archetype, each drawn from the same source data but reframed for that audience's "cares about / reacts badly to / lead with" profile.

## Preconditions

- `CHARTER.md` at revision ≥ 1
- `STAKEHOLDERS.md` populated
- At least one weekly status in the last 14 days

## Intent: create

1. **Choose archetypes.** Show the audience pool from `references/audience-profiles.md`:
   - CIO / CTO / CFO / COO / Client-PM / Engineering-Lead / Legal-Compliance / Board-NED / Executive-Sponsor
2. **Multi-select.** User can pick 1–N archetypes. *"Generate updates for which roles? Comma-separated or all."*
3. **Source.** Read most recent weekly status, RAID top-5, recent decisions, open change requests, charter Success Criteria.
4. **Per archetype, compose** in this structure:

```markdown
---
artifact_type: stakeholder-update
engagement: <slug>
date: <YYYY-MM-DD>
audience_archetype: <CIO | CFO | ...>
generated_by: pdf-write-stakeholder-update
red_teamed: false
sources:
  - 04-execution/weekly-status/<YYYY-Www>.md
  - 04-execution/RAID.md
  - decision-log.md
---

# Update for the <Archetype> — <Engagement name> — <date>

**Headline:** <one sentence; tuned to what this archetype cares about most>

## <Lead-with section, tuned to the archetype>

<2–4 bullets. The audience profile determines what leads. CFOs lead with budget/burn; CIOs with risk/security; COOs with operational impact; CTOs with architecture/quality; Legal with compliance; etc.>

## Where we are vs. plan

<one paragraph. Honest. Reconciles to charter Success Criteria.>

## What you should know

<2–4 bullets. Material developments since last update relevant to this archetype.>

## What we need from you

<0–2 items. Specific. Each: ask, why, by when.>

## Coming up that affects you

<1–2 bullets. Forward look filtered to this archetype's domain.>

---

_Generated by `pdf-write-stakeholder-update`. Reconciles to weekly status <YYYY-Www>, charter rev <N>._
```

5. **Run simulated red-team per file.** Three archetype-specific cynical-review questions (see `references/audience-profiles.md` for the per-archetype red-team prompts).
6. **Confirm before write.** Show all generated drafts; allow per-draft accept/refine/skip.
7. **Write each accepted draft** to its own file.
8. **Hand off:** if multiple archetypes were generated and some were skipped, offer to re-run with refined framing.

## Intent: update

1. Identify the existing update by date and archetype.
2. Same diff-and-confirm flow.
3. Re-run red-team on every changed file.

## Intent: validate

Per file:
- [ ] Frontmatter has `audience_archetype` from the allowed set
- [ ] Lead-with section matches the archetype's "lead with" rule (validated against `references/audience-profiles.md`)
- [ ] "What we need from you" entries each have ask + why + by-when (or section says "Nothing")
- [ ] No content from one archetype appears verbatim in another's lead-with section
- [ ] Frontmatter `red_teamed: true`

## Intent: dump-merge

Accept a draft from dump flow. Try to infer the archetype from the content; if unclear, ask the user. Then compose into the canonical structure for that archetype.

## Anti-patterns to refuse

- **Single update sent to multiple archetypes.** That is what `pdf-write-exec-summary` is for. If the user resists per-archetype framing, route them to exec-summary.
- **Same lead-with paragraph across archetypes.** Validation flags this. Fidelity, not flattery.
- **Framing the same news as positive to one archetype and negative to another.** Honest framing means the underlying *judgement* of how things are going stays consistent across archetypes; what changes is emphasis, not truth.

## Red-team posture

**Default ON, per file.** Each generated update gets its own red-team pass with archetype-specific cynical-review questions.
