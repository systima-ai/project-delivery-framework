# Escalation memo workflow

Produces `_pdf-output/engagements/{active}/05-governance/escalations/<YYYY-MM-DD>-<slug>.md`.

**Structure: SCQA** (Situation, Complication, Question, Answer). See `references/scqa-guide.md` for the full method. The memo exists for one purpose: to extract a decision from a named person by a named date. Helena's principle 6 is the constraint — this is not a venting document or a status update.

## Preconditions

- Active engagement
- At least one named stakeholder in `STAKEHOLDERS.md` who can be the recipient
- Usually triggered by:
  - A blocker file with deadline approaching (`pdf-triage-blocker` offers Helena)
  - A risk that has materialised (`pdf-update-raid` close intent on a materialised risk offers Helena)
  - A change request that needs an above-the-line approver (`pdf-create-change-request` from Klaus)
  - The user invoking Helena directly

## Intent: create

1. **Identify the recipient.** Ask: *"Who is this memo addressed to? (Single named individual — escalations are not committee documents.)"* Validate against STAKEHOLDERS.md. If the recipient isn't there, prompt to add them first.
2. **Identify the trigger artifact, if any.** Ask: *"Is this triggered by a blocker, a materialised risk, a change request, or none of those?"* If a specific file, read it for context.
3. **Walk SCQA.** One section at a time:

   **Situation:**
   - *"In one paragraph, what is true that everyone already agrees on? The baseline reality the recipient won't dispute."*
   - Reject if the answer contains anything the recipient might dispute. Situation is the agreed ground only.

   **Complication:**
   - *"What has changed, gone wrong, or threatens — that the recipient doesn't yet fully see?"*
   - This is the "but" of the memo. One paragraph; specific; named.

   **Question:**
   - *"In one sentence, what is the implicit question the recipient is now asking — or should be asking?"*
   - Pyramid principle. If you cannot state the question in one sentence, the memo isn't ready.

   **Answer:**
   - *"What is your recommendation? And what specific decision do you need from the recipient, by when?"*
   - Required fields: recommendation (one paragraph), decision-needed (one sentence), deadline (a date), what-happens-without-decision (one sentence).
4. **Tone check.** Run a deliberate pass:
   - Is there blame language? Strip it.
   - Is there hedging that obscures the recommendation? Sharpen.
   - Does the memo read as venting rather than asking? Refactor.
5. **Compose:**

```markdown
---
artifact_type: escalation-memo
engagement: <slug>
date: <YYYY-MM-DD>
addressed_to: <name>
trigger: <blocker BLK-NNN | risk R-NNN | change CR-NNN | none>
decision_needed_by: <YYYY-MM-DD>
generated_by: pdf-write-escalation-memo
red_teamed: false
tone_check_pass: false
---

# Escalation — <one-line subject> — <date>

**To:** <name>, <role>
**From:** <author>
**Decision needed by:** <date>
**Trigger:** <reference if any>

## Situation

<one paragraph; agreed ground only>

## Complication

<one paragraph; what has changed>

## Question

<one sentence; the implicit question>

## Answer

**Recommendation:** <one paragraph>

**Decision I need from you:** <one sentence>

**By when:** <date>

**Without this decision:** <one sentence on consequence — cost, slip, risk materialisation, etc.>

---

_Reconciles to <trigger artifact>. Generated by `pdf-write-escalation-memo`._
```

6. **Run simulated red-team inline:**
   - *Is the question genuinely the recipient's question — or is it mine?*
   - *Is the recommendation honest, or self-serving?*
   - *Would a reasonable recipient feel ambushed by this memo?*
   - *Is the deadline real, or arbitrary?*
   Apply revisions.
7. **Run tone check.** A second pass dedicated to language: blame, hedging, venting. Update `tone_check_pass: true` in frontmatter when clean.
8. **Write the file.**
9. **Hand off:**
   - Offer to draft a covering Teams / email message for sending the memo.
   - Offer `pdf-decision-log` to record the decision when it arrives.
   - Append to STATUS-LOG with a one-line pointer.

## Intent: update

1. Identify the memo by date/slug.
2. Common updates: decision received → log outcome; deadline extended → revise; new information → revise Situation/Complication.
3. Re-run red-team and tone check on any substantive revision.
4. The prior file is renamed `.v<n>.md`.

## Intent: validate

- [ ] Frontmatter: `addressed_to` is a named individual (single name, not a group)
- [ ] `decision_needed_by` is a real date in the future (or the memo is in `update` for a decision received)
- [ ] All four SCQA sections present and non-empty
- [ ] Situation contains nothing the recipient would dispute (cynical-review judgement)
- [ ] Question is exactly one sentence
- [ ] Answer section has recommendation, decision-needed, deadline, without-decision-consequence
- [ ] Frontmatter `red_teamed: true` AND `tone_check_pass: true`

## Intent: dump-merge

Accept a draft or rant from Helena's dump flow. Reshape into SCQA. The most common dump pattern: user pastes a long frustrated message; the workflow must:
1. Extract Situation (what's agreed).
2. Extract Complication (what's wrong) — strip emotional adjectives but preserve the substance.
3. Identify the buried Question.
4. Identify the buried Answer / recommendation.
5. Surface what's missing (recipient? deadline? consequence?) and elicit those.

## Anti-patterns to refuse

- **Memos addressed to a group.** Escalations have one decision-maker. If multiple people need the same memo, generate one per recipient.
- **No deadline.** Refuse to finalise without `decision_needed_by`.
- **No "without this decision" line.** This is the consequence statement that turns a request into an escalation. Without it, the memo is just a question.
- **Blame language.** Strip on tone check. "X has failed to deliver" → "Y is not on schedule because Z" (named cause, not named culprit, unless naming the culprit is the whole point — and if so, name and own it).
- **Venting.** The memo's purpose is decision-extraction. If the draft reads as venting, reshape ruthlessly. Helena's principle 6.

## Red-team posture

**Default ON.** Plus the tone-check pass (specific to escalation memos because the failure mode is voice, not just substance).
