# Translate workflow

Rewrites a passage for a target audience. Two translation classes (per Lloyd's B choice):

| Class | Examples |
|---|---|
| **Audience-up** | engineering-detail → CIO; tactical-status → board-level; commercial-numbers → CFO |
| **Peer (lateral)** | commercial ↔ engineer; legal ↔ business; technical ↔ board; product ↔ delivery |

See `references/translate-archetypes.md` for the full archetype list and per-archetype framing rules.

## Preconditions

- Source text (paste or path)
- Target audience identifier (single archetype) — and optionally a source audience (helps the translation)

## Intent: translate (default)

1. **Establish inputs.**
   - *"Paste the source text, or give me a path."*
   - *"Target audience? (List: CIO, CTO, CFO, COO, Client-PM, Engineering-Lead, Legal-Compliance, Board-NED, Executive-Sponsor, Commercial-Lead, Product-Lead, Engineer.)"*
   - *"Source audience, if helpful — or 'unspecified'."*

2. **Read the source.** Identify:
   - What's being said (the substance)
   - Who it sounds written for (the implicit current audience)
   - What technical / domain language is used
   - What's foregrounded vs backgrounded

3. **Translate.** Apply the target archetype's framing (from `references/translate-archetypes.md`):
   - Re-order the substance — what the target archetype cares about most leads
   - Re-language — replace domain language the target doesn't use
   - Re-scope — surface what the target needs to act / decide; background what they don't

4. **Preserve substance integrity.** The translation must not change the underlying claim. If the source says "we're behind schedule by 2 weeks", the translation says the same — in the language and framing the target reads. Translation that sanitises bad news into ambiguity is **mistranslation** and the workflow refuses it.

5. **Compose the output:**

```markdown
---
artifact_type: translation
date: <YYYY-MM-DD>
source_audience: <archetype or "unspecified">
target_audience: <archetype>
source_path: <path or "inline">
generated_by: pdf-translate
substance_preserved: true
---

# Translation — <source audience or "unspecified"> → <target audience>

## Source (verbatim)

> <source text>

## Translation

<translated text — same substance, target archetype's framing>

## Translation notes

- **What I re-ordered:** <one or two bullets — what now leads vs what was buried>
- **What I re-languaged:** <one or two bullets — which terms changed>
- **What I re-scoped:** <one or two bullets — what's now foreground vs background>
- **What I deliberately preserved:** <one bullet — the substance commitment that must not move>

## Substance check

- Claims in source: <list of substantive claims>
- Claims in translation: <same list>
- Match: ✓ (no claims dropped or softened)

---

_Generated by `pdf-translate`._
```

6. **Hand off.**
   - The translation is the output; the user typically pastes it into a downstream artifact (a stakeholder update, an exec summary, a 1:1 brief).
   - If the source text was already in a PDF artifact, offer to update that artifact's `red_teamed` posture (a freshly-translated section may need a fresh red-team).

## Intent: translate-with-feedback

Same flow, but at the end the workflow runs a self-critique: *"What would the target archetype find most challenging or least clear in this translation? Did the translation accidentally soften a claim?"* Surface to the user; iterate.

## Intent: validate

- [ ] Source and target audiences declared
- [ ] Translation present
- [ ] Translation notes present (the four "what I" bullets)
- [ ] Substance check populated; claims match
- [ ] Frontmatter `substance_preserved: true`

## Intent: dump-merge

Effectively the same as default for this skill — translation is inherently a transform-on-paste.

## Anti-patterns to refuse

- **Translation that softens a hard claim.** Refuse. "We're 2 weeks behind" must remain a hard claim in any translation; only the surrounding framing changes.
- **Translation that loses a claim entirely.** Refuse. Translation is not summarisation.
- **Translation that smuggles new claims in.** The translation cannot add commitments, claims, or framing the source didn't make.
- **Target archetype not on the supported list AND no profile provided.** Workflow asks the user to either name a supported archetype or supply a one-paragraph profile of the target (which it will then use as an ad-hoc archetype).
