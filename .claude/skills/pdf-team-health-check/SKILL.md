# Team health check workflow

Produces `_pdf-output/engagements/{active}/09-people/team-health-checks/<YYYY-MM-DD>.md`. **Confidential.**

This is the **aggregate** view — team-level signal, not individual. Individual-level signals live in `pdf-attrition-risk`, `pdf-prep-1on1`, and `pdf-prep-performance-conversation`.

## Preconditions

- Active engagement
- `STAKEHOLDERS.md` populated

## Intent: create

1. **Establish source.** *"Have notes / retro export / pulse-survey export to ingest? Or shall we go by your read of the team this week?"* Same paths-or-interview pattern as Quinn's workflows.

2. **Walk the five aggregate dimensions:**

   - **Morale** — overall mood, recent shifts, what's driving it
   - **Capacity** — sustainable load? Overtime patterns? Anyone showing burnout signal?
   - **Blockers (team-level)** — what's slowing the team down at the team level? (Individual blockers go via `pdf-triage-blocker`.)
   - **Growth and learning** — is anyone visibly stretched, stalled, or under-utilised?
   - **Team dynamics** — collaboration patterns; any tension; any quiet voices

   For each: ask the canonical question, capture observations, assign a RAG (Green / Amber / Red), record what evidence supports the rating.

3. **Compose:**

```markdown
---
artifact_type: team-health-check
engagement: <slug>
date: <YYYY-MM-DD>
confidential: true
overall_rag: <R|A|G>
sources: [<paths> or "interview"]
generated_by: pdf-team-health-check
---

# Team health — <date>

> Confidential. Aggregate signal only; individual signals live elsewhere.

## Summary

| Dimension | RAG | Headline |
|---|:-:|---|
| Morale | <G/A/R> | <one line, aggregate only> |
| Capacity | <G/A/R> | <one line> |
| Blockers (team) | <G/A/R> | <one line> |
| Growth and learning | <G/A/R> | <one line> |
| Team dynamics | <G/A/R> | <one line> |
| **Overall** | **<G/A/R>** | **<one line>** |

## Narrative

### Morale — RAG: <colour>

<paragraph; aggregate observations only. Never name individuals here.>

**What might help:** <one line>

(Repeat per dimension.)

## Trend (if multiple checks exist)

(Brief; the last 3 checks; movement per dimension.)

## Actions

- <action — owner — by when>
- ...

## Cross-references

- Recent attrition watch: <path or "—">
- Recent ramp plans: <path or "—">
- Specific individuals to focus 1:1 prep on: <referred elsewhere — do NOT name here>

---

_Generated by `pdf-team-health-check`. Confidential. Next default check: <date 14 days out>._
```

4. **Refuse to name individuals** in the narrative or the summary. If an issue is genuinely person-specific, the workflow flags this and routes the user to the relevant individual workflow:
   - Attrition risk → `pdf-attrition-risk`
   - 1:1 specific to one person → `pdf-prep-1on1`
   - Performance concern → `pdf-prep-performance-conversation`

5. **Cross-skill offers:**
   - Any AMBER+ in Capacity → recommend `pdf-attrition-risk` if not refreshed in 4 weeks
   - RED Morale or Team dynamics → recommend `pdf-prep-1on1` with two or three named individuals (the user names; Iris doesn't pick)
   - RED overall → flag in the next `pdf-weekly-status` Asks section *as an aggregate signal* — never with names

6. **Write the file.**

## Intent: update / validate / dump-merge

Same shape as Quinn's health-check workflows. Cadence: every 14 days by default (engagement-overridable).

## Anti-patterns to refuse

- **Naming individuals.** This workflow is aggregate-only. If the user starts naming, the workflow gently redirects: "Looks like this is person-specific; want to route to `pdf-prep-1on1` for that person, and keep this check at aggregate level?"
- **GREEN overall with thin evidence.** If sources are "interview" and the narrative is one line per dimension, the workflow asks for more substance before finalising.
- **Same headline finding three checks in a row.** The workflow flags this pattern and asks if any action has been taken since the last check — health-check repetition without action is the underlying disease.

## Red-team posture

Off. Confidential operational artifact.
